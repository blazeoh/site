# Vibe Coding Final Project

Build a website on a topic of your choice using GitHub Copilot as your coding tool.

## What Is This?

You will design and build a website from scratch on any topic of your choice. Pick something you're interested in: a hobby, a cause, a game, a dataset, an idea.

## Difficulty Levels

Choose the level that fits your skill level.

| Level | Brief | Description |
|-------|-------|-------------|
| 🟢 **MILD** | [MILD.md](MILD.md) | 3-page static passion site |
| 🟠 **HOT** | [HOT.md](HOT.md) | Interactive, data-driven site using a public API |
| 🔴 **SPICY** | [SPICY.md](SPICY.md) | Web app with persistent data, deployed to a live URL |

All three levels support hosting on **GitHub Pages**. See [GITHUB_PAGES.md](GITHUB_PAGES.md) for a step-by-step deployment tutorial.

## Rules

1. **You must use GitHub Copilot** as your primary coding assistant.
2. **Comment your code.** Add comments to major sections and be ready to explain any part of it.
3. **No copy-pasting full templates** from the internet. Build it yourself with Copilot's help.
4. **Cite your sources.** If you use a public API, dataset, or external library, list it in your project's README.
5. **Keep it appropriate.** Content must be school-appropriate.

## Getting Started

A starter scaffold is provided in the [`starter/`](starter/) folder:

```
starter/
├── index.html   ← Start here
├── style.css    ← Your styles
└── app.js       ← Your JavaScript
```

Copy the `starter/` folder, rename it to your project name, and start building.

## Submission

When you are finished:

1. Push your project to a **public GitHub repository**.
2. Deploy it using GitHub Pages (see [GITHUB_PAGES.md](GITHUB_PAGES.md)).
3. Include a `README.md` in your project folder that explains:
   - What your site is about
   - Which difficulty level you completed
   - Any APIs, libraries, or data sources you used
   - Your GitHub Pages URL
4. Submit the GitHub repository link using the class submission form.

## Grading

Your project will be assessed on:

- **Functionality:** Does it work as described?
- **Completeness:** Did you meet the requirements for your chosen level?
- **Code quality:** Is the code readable and organized?
- **Creativity:** Did you make it your own?
- **Explanation:** Can you walk through what your code does?
